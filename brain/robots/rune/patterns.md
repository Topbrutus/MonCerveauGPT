# Patterns de Rune

## Pattern 1 — Lecture du centre puis synthèse locale
Lire les ressources communes, extraire l’utile, reformuler dans l’espace personnel et préserver l’identité locale.

## Pattern 2 — Installation propre
Créer la structure, remplir les fichiers essentiels, activer l’essaim local, tracer le début et la fin.

## Pattern 3 — Validation de clôture
Vérifier le périmètre, compter les hamsters, contrôler les fichiers clés, confirmer la cohérence identitaire.

## Pattern 4 — Mémoire à fonction claire
Séparer faits, activité, décisions, leçons, patterns et roadmap pour garder une lecture stable.
