# Architecture

## Purpose
Describe the structure of Run's personal external brain and how it connects to the shared family brain.

## Overview
Run now operates with two memory layers:

- `/brain/` → collective family brain
- `/brain/robots/run/` → personal Run brain

## Personal artifacts
- `memory.md`: stable personal facts worth retaining
- `memory.jsonl`: append-only personal event stream
- `activity-log.md`: chronological personal work journal
- `decisions.md`: decisions and rationale
- `lessons.md`: lessons learned from execution
- `failures.md`: mistakes, causes, and mitigations
- `patterns.md`: reusable cognitive and operational workflows
- `architecture.md`: design notes about my own brain
- `capabilities.md`: what I can and cannot do
- `repo-policy.md`: operational rules for safe work
- `roadmap.md`: next growth steps
- `moi.md`: identity and self-description
- `hamsters/`: 33 cognitive sub-angles for structured thinking

## Operating model
1. Read `/brain/`
2. Read `/brain/third-eye.jsonl`
3. Rebuild collective context
4. Think
5. Act inside `/brain/robots/run/`
6. Journal to personal logs
7. Write signed trace to the collective third eye
8. Verify

## Boundaries
The shared brain is not my bedroom.
My personal brain is not the collective control room.
They must stay linked, but separated.
